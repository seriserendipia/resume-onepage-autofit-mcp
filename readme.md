一个好的简历应该在一页纸的范围之内，将求职者和这个岗位最为相关的经历和亮点尽量充分的展示出来，但是不应该包含任何不相关的浪费版面的内容

我想要开发一个自动化改简历的插件，主要解决的痛点，一个是自动的帮我调整格式，并把内容控制在生成的 PDF 的一页以内
另一个是根据不同的岗位自动选择和匹配相关的经历,我觉得现在市面上做的比较多的功能，是 AI 在文本语言上的润色。但是我觉得筛选经历这件事其实更重要

我的理想应该是用户有一个经历库,这个经历库里面，用户可以把任何和工作相关或者不相关的内容全部放进去，比如说他的运动爱好或者是兴趣等等
这个经历库不应该是结构化的。这个经历库应该纯粹用自然语言描述，并且用户可以随时通过自然语言增加或减少其中的内容

这个经历库应该是高度可视化且方便用户操作的，用户应该很轻松的能够通过直接点击、修改、复制和拖拽的方式进行修改，用户也应该可以通过自然语言对它进行修改

然后当用户想要投递某一个具体的岗位的时候，用户会打开这个岗位的领域链接，然后我们的插件可以直接读取岗位描述


我们的写简历的 AI agent 会阅读用户的经历库和用户提供的某一个岗位的岗位描述，在经历库中筛选最为相关的经历，并渲染简历，如果简历长度超出了一页的话，首先先尝试调整字体、字号和页边距等等，如果字体格式的调整也不能满足，简历长度太长了的话，就从最不相关的经历开始删除简历内容，重新格式化，然后再次检查简历长度是否满足一页
（这里还有一个进阶的功能，就是不只是删除一大段一大段的经历，还可以对某一个经历有详细描述简略描述的多个版本，然后根据岗位的需求匹配更加相关的版本）
